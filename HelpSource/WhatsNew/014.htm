Версия 7.0.014 (29 июля 2005)

Содержание <<Предыдущая Следующая>>  

Общая информация

  • Пенсионный отчёт - теперь значения процентов для ФОТа берутся из соответствующих кодов ФОТ
  • В документе Табель (Оклад) появилась возможность вводить сумму премии вручную
  • Форма 1ДФ - на двух закладках в строке итога и столбце ФИО уменьшен и изменён шрифт
  • Сделано добавление параметра в пользовательские таблицы - "Максимальный доход", для тех у кого его не было. Перед работой, после обновления - заполните его. На сегодняшний день он равен 4100
  • Добавлен запрос в базу данных - GetDepends. Используется в отчётах. Мог присутствовать не у всех
  • Добавлено свойство для корреспондентов "Премия, %". Если заполнено, то в форме Табель (Оклад) будет заполнен столбец Премия значением равным проценту от оклада
  • Форма 1ДФ - убран 27 признак дохода
  • Форма 1ДФ - физическим лицам присвоен код признака дохода равный 42
  • Форма 1ДФ - исключена ситуация когда в файл DBF не записывалась последняя строка (не корреспондент) из формы
  • Форма 1ДФ - исправлена ошибка, когда в файле DBF были перепутаны местами итоговые столбцы
  • При открытии Базы Данных отображается диалоговое окно с информацией о названии настройки, её текущей и оригинальной (до обновлений) версии
  • Форма 1ДФ - расширен список признаков дохода до 57

Формы

  • Изменена форма Табель (Оклад) для работы со свойством Премия, % и вводом премии вручную
  • Изменена форма 1ДФ. Исправление ошибок и новое законодательство

Диалоги

  • Изменён диалог dbf_1df.adg для корректого формирования файла dbf
  • Добавлен диалог version.adg для отображения сведений о текущей настройки при открытии Базы Данных

Модули

  • Добавлен служебный модуль workarea.avb, который при открытии Базы Данных выводит диалог version.adg

Инструкция

  • После обновления проверьте значение параметра Максимальный доход в пользовательских таблицах. Если оно пусто, то введите туда 4100 и начинайте работать.
