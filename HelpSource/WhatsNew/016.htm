Версия 7.0.016 (4 октября 2005)

Содержание <<Предыдущая Следующая>>  

Общая информация

  • Добавлен отчёт ФОТ по подразделению
  • Исправлена ошибка в Табель (оклад). Появлялась при заполнении подразделения, в котором есть хоть один человек, работающий по тарифу
  • Расширена функциональность формы Табель (оклад)
  • Добавлены новые формы Больничные по беременности, Больничные по уходу за ребёнком, Больничные по травме, Выходное пособие, Увольнительные
  • Изменены 20 методов, для оповещение о внутренних ошибках расчёта

Формы

  • Табель (оклад) - теперь есть выбор - заполнять подразделение для расчёта в часах или в днях.
  • Табель (оклад) - появилась возможность выбора корреспондентов по табельному номеру
  • Табель (оклад) - для каждого столбца начислений выводится итоговая сумма
  • Табель (оклад) - исправлена ошибка, когда при заполнении подразделения, в котором работают как по окладу так и по тарифу, не выполняется расчёт
  • Больничные по беременности - новая форма
  • Больничные по уходу за ребёнком - новая форма
  • Больничные по травме - новая форма
  • Выходное пособие - новая форма
  • Увольнительные - новая форма
  • Прим: во всех новых формах автоматизирован расчёт средних показателей

Отчёты

  • Добавлен отчёт: ФОТ по подразделению. Необходимо подключить на необходимое подразделение перед использованием

Запросы

  • Добавлен запрос: DependsPerSquadPerAgentInPpLowerMax. Используется в отчёте ФОТ по подразделению
  • Добавлен запрос: GetFotIDs. Используется для получения коллекции методов ФОТ

Методы

  • Изменено 20 методов. Теперь во время расчёта, при нехватке данных для расчёта будет выводится предупреждение в информационное окно (внизу экрана)
  • Добавлены новые методы для работы новых форм. Названия методов, такие же как у форм. В них применяется расчёт средних показателей, что бы он работал - надо добавить в зависимость коды, из суммы которых будет формироваться среднее значение.

Инструкция

  • Отчёт ФОТ по подразделению - перед работой надо подключить к необходимому подразделению.
  • Важно! Новые коды автоматически не добавляются в зависимости. Добавьте их вручную в зависимость кодов, отвечающих за расчёт налогов.
